# **tg_bot_HB**

Telegram-приложение для хранения и управления днями рождения и праздниками.  
Архитектура: **Telegram Bot + Mini App**.

---

## **Как работает приложение**

Приложение состоит из отдельных компонентов с чётким разделением ролей.

---

## **Поток работы пользователя**

1. Пользователь открывает Telegram-бот  
2. Выполняет команду `/start`  
3. Бот отправляет кнопку открытия Mini App  
4. Mini App открывается внутри Telegram  
5. Mini App взаимодействует с Backend API  
6. Backend читает и записывает данные в PostgreSQL  

---

## **Telegram Bot**

### **Назначение**

Telegram Bot используется только как точка входа.

### **Что делает**

- Обрабатывает команду `/start`
- Отправляет кнопку открытия Mini App

### **Чего не делает**

- Не хранит данные  
- Не содержит бизнес-логики  
- Не управляет состоянием пользователя  

---

## **Telegram Mini App**

### **Назначение**

Mini App — основное пользовательское приложение.

### **Функции**

- Отображение календаря
- Добавление и редактирование событий
- Загрузка данных пользователя
- Отправка запросов в Backend API

Mini App работает внутри Telegram WebView.

---

## **Backend API**

### **Назначение**

Backend реализует серверную логику.

### **Функции**

- Приём HTTP-запросов
- Валидация данных
- Работа с PostgreSQL
- Формирование JSON-ответов

---

## **База данных**

### **Хранилище**

PostgreSQL используется как основное хранилище данных.

### **Содержит**

- События
- Даты
- Пользовательские идентификаторы
- Служебные поля

---

## **Запуск проекта**

### **Docker Compose**

Проект разворачивается через Docker Compose.

Порядок запуска сервисов:

1. PostgreSQL  
2. Backend API  
3. Frontend (Mini App)  
4. Nginx  
5. Telegram Bot
